## Introduction

When you use a website or an app, everything feels simple.

You open a page, click a button, log in, or view some content. Things respond quickly, and you rarely think about what is happening behind the scenes.

But every one of those interactions depends on multiple parts working together.

Some of the code is responsible for what you see and interact with. Other parts run somewhere else, handling logic, storing data, and deciding what should happen when you do something.

Web development is about building these systems so that everything works together as one application.

## Thinking in Systems

At a high level, every web application follows the same basic idea.

There is a part that the user interacts with, and there is a part that handles the actual work behind the scenes. These parts communicate with each other to produce the final result.

You can think of it as a system rather than a single piece of code.

Even a simple application needs to handle input, process it, and return something back. As applications grow, this becomes more complex, but the core idea stays the same.

---

## Frontend and Backend

The frontend is everything you see in your browser. It is responsible for displaying information and handling user interaction. When you click a button, type into a form, or navigate between pages, that is happening in the frontend.

The backend is where the actual work happens. It runs on a server and is responsible for handling requests, applying logic, and deciding what data should be returned. If something needs to be saved or retrieved, the backend is involved.

These two parts are separate, but they depend on each other. The frontend needs data from the backend, and the backend needs requests from the frontend to know what to do.

---

## APIs and Databases

For the frontend and backend to communicate, they use something called an API. This is simply a way for one part of the system to ask another part for something.

For example, the frontend might ask the backend for a list of lessons or send a request to mark something as completed. The backend processes that request and sends back a response.

Most applications also need a place to store data. This is where databases come in. Things like users, content, and progress are stored there so they can be retrieved later.

You do not need to know how these are implemented yet. For now, it is enough to understand the role they play in the system.

---

## Fullstack Development

Some developers work only on one part of the system, but others work across the entire application. These are called fullstack developers.

A fullstack developer understands how the frontend, backend, and data layer fit together. They can move between these parts and build features that connect everything into a working system.

This course follows that path. You will start by learning how the backend works, and later you will connect it to a frontend to build complete applications.
